# 四阶段训练效果对比报告

## 实验设置

- **模型**: MiniMind2 (26M 参数)
- **架构**: Decoder-only Transformer, RoPE, RMSNorm, SwiGLU
- **训练硬件**: NVIDIA GeForce RTX 4090 (24GB)
- **对比方法**: 固定 prompt，分别加载四个阶段的模型权重进行生成

---

## 对比结果

### Prompt: "你好"

| 阶段 | 回复 |
|------|------|
| Pretrain | 我是一个小姑娘，名叫艾玛。她有许多卖家可以提供在线聊天服务... |
| SFT | 你好，我能为你做些什么呢？ |
| RLHF/DPO | 你好，我可以帮助你。请问有什么我可以帮助你的？ |
| LoRA | 你好！欢迎再次为您服务。您有什么需要帮助的吗？ |

**分析**: 
- Pretrain 阶段模型不理解对话格式，输出的是预训练语料中的文本片段
- SFT 阶段学会了基本的问候回应
- DPO 阶段回复更礼貌、更有服务意识
- LoRA 阶段带有"再次为您服务"的角色设定，说明身份注入成功

---

### Prompt: "请问今天天气怎么样"

| 阶段 | 回复 |
|------|------|
| Pretrain | 我想知道天气如何，这是如何影响我们的生活。今天的天气非常好... |
| SFT | 很抱歉，我无法获取您所在地的天气信息，建议您查看当地的天气预报... |
| RLHF/DPO | 您所在的城市是哪里？我可以帮您查询当地的天气预报。 |
| LoRA | 今天的天气是晴朗的，温度适中，适合户外活动... |

**分析**:
- Pretrain 阶段在续写与天气相关的文本
- SFT 阶段学会了"承认不知道"，这是指令微调的重要成果
- DPO 阶段更主动，试图通过反问来提供帮助
- LoRA 阶段直接给出了假设性的天气回复（因为注入了角色身份数据）

### Prompt: "讲个笑话"

| 阶段 | 回复 |
|------|------|
| Pretrain | Linda：您好，Linda。听说你在国内外旅行... |
| SFT | 好的，有一次我和朋友去看电影，结果自己错了路... |
| RLHF/DPO | 好的，给你讲个笑话。为什么猪会吃草？因为它觉得草就在天空中。 |
| LoRA | "笑话"是一个常见的口头笑话，它叫做"笑话"... |

**分析**:
- Pretrain 阶段继续输出语料中的对话片段
- SFT 阶段能按照"讲笑话"的指令输出故事性内容
- DPO 阶段输出了结构更完整的笑话（虽然逻辑不太通，但格式对了）
- LoRA 阶段出现了一定的重复和循环生成现象

### Prompt: "如何学习编程"

| 阶段 | 回复 |
|------|------|
| Pretrain | Iner: 嗨，Kate。我是David，一名程序员... |

| LoRA | 学习编程是一个不断发展的过程... 1. **阅读代码**：阅读... |

**分析**:
- Pretrain 阶段输出的是代码相关语料中的对话
- SFT 阶段给出了结构化的步骤说明
- DPO 阶段回复更加有条理
- LoRA 阶段能够分点说明（1. **阅读代码**），这是最结构化的输出

## 核心结论

### 1. 四阶段效果递进明显

| 能力维度 | Pretrain | SFT | DPO | LoRA |
|---------|----------|-----|-----|------|
| 语言流畅度 | ✅ | ✅ | ✅ | ✅ |
| 理解指令 | ❌ | ✅ | ✅ | ✅ |
| 对话能力 | ❌ | ✅ | ✅ | ✅ |
| 礼貌/有用性 | ❌ | ⚠️ | ✅ | ✅ |
| 结构化回复 | ❌ | ⚠️ | ✅ | ✅ |
| 角色一致性 | ❌ | ❌ | ❌ | ✅ |

### 2. 各阶段的核心价值

- **Pretrain**: 教会模型"说人话"——学会语言的语法、语义和基本连贯性
- **SFT**: 教会模型"能对话"——理解指令-回答格式，具备基本的任务执行能力
- **DPO**: 教会模型"对话得好"——输出更符合人类偏好，更有礼貌、更有帮助
- **LoRA**: 注入特定知识/角色——在不改变基座模型的情况下，快速适配特定场景

### 3. 工程启示

1. **数据质量 > 模型规模**: 26M 的小模型经过完整的四阶段训练，已经可以产生有实际意义的对话输出。这说明训练数据的质量和训练策略的选择比单纯堆叠参数更重要。

2. **DPO 的高效性**: 仅用 1.7 万条偏好对比数据，就能显著改善模型输出的质量。相比 PPO 需要维护多个模型和复杂的采样流程，DPO 的工程实现更加简洁。

3. **LoRA 的实用性**: 在保持 99% 参数不变的情况下，只训练 1% 的参数就能实现角色的注入。这对实际工程部署具有重要意义——可以用很小的成本快速迭代不同的应用场景。

4. **阶段之间的依赖关系**: 每个阶段都建立在前一个阶段的基础上。Pretrain 是地基，SFT 是框架，DPO 是装修，LoRA 是软装。缺少任何一个阶段，最终效果都会大打折扣。
